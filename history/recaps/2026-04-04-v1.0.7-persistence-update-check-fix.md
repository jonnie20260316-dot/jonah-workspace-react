## What Was Worked On

### Lane 1: Data Persistence Stability (3 critical fixes)

**Problem 1: GitHub settings lost across quit/restart**
- Root cause: `github-sync-enabled`, `github-sync-repo`, `github-sync-token` were not added to `GLOBAL_KEYS` allowlist (JW-8 violation from v1.0.6)
- Consequence: Keys routed to `session:{activeDate}:{key}` paths; on boot with empty `_activeDate`, settings unreadable
- Fix: Added 3 keys to `GLOBAL_KEYS` array in `src/constants.ts`
- Boot migration: Added migration block in `src/hooks/useSyncBoot.ts` to rescue stranded session-scoped keys before `store.initDeviceId()`

**Problem 2: "立即同步" (immediate sync) button stops active video recordings**
- Root cause: `githubSyncNow()` → `rehydrateStores()` → `setState({ blocks: newArray })` with fresh array reference triggered React unmount of VideoCaptureBlock → `track.stop()` on all MediaStreams
- Fix: Implemented granular block merge in `rehydrateStores()` — compare block IDs/order, merge changed properties per-block instead of replacing array
- Same-device skip: Added deviceId check in `githubSyncNow()` to skip rehydration entirely if pulling own data back from GitHub

**Problem 3: macOS screen recording permission reset on update**
- Root cause: OS-level constraint — TCC ties permissions to app code signature CDHash; ad-hoc signing changes per update → permission revoked
- Fix: Added permission status detection + bilingual user guidance banner in `VideoCaptureBlock.tsx`
- Implementation: IPC handler `app:screen-permission-status` → `systemPreferences.getMediaAccessStatus('screen')`

**Files changed:**
- `src/constants.ts` — added 3 keys to GLOBAL_KEYS
- `src/hooks/useSyncBoot.ts` — boot migration for stranded session-scoped keys
- `src/stores/useSyncStore.ts` — granular block merge + same-device skip
- `src/utils/sync.ts` — return payload from `githubPull()`
- `src/types.ts` — added `getScreenPermissionStatus()` to electronAPI
- `electron/main.cjs` — IPC handler for screen permission status
- `electron/preload.cjs` — exposed `getScreenPermissionStatus()`
- `src/blocks/VideoCaptureBlock.tsx` — permission check + guidance banner

### Lane 2: Update Checker Fix (broken feature)

**Problem: "Check Updates" button was broken**
- Root cause: `electron-updater` with `forceDevUpdateConfig=true` required `dev-app-update.yml` to exist in project root, but file was never created
- Secondary: Error reporting was silent — `updater:check` handler caught exceptions but didn't pass message to UI
- Consequence: Users couldn't check for updates; silent failure with no feedback

**Fix:**
1. Created `dev-app-update.yml` with GitHub publisher config (owner, repo, provider, cache name)
2. Added `dev-app-update.yml` to `electron-builder.yml` files list so it's bundled in packaged app
3. Modified `electron/main.cjs` `updater:check` handler to capture and pass error message to UI

## Key Decisions

1. **Boot-time migration over lazy migration:** Migrated stranded session keys immediately on init rather than on-first-read. Ensures consistent global path from boot onward.

2. **Granular merge over full replace:** Instead of replacing entire block array on sync, compared block IDs/order and merged only changed properties. Preserves React component identity and prevents unmount of recording blocks.

3. **Same-device skip:** When pulling own data back from GitHub, skip `rehydrateStores()` entirely. localStorage already correct; rehydrate only for cross-device sync.

4. **dev-app-update.yml as separate file:** Created explicit YAML config rather than embedding in electron-builder.yml. Electron-updater expects this file to exist in app root at runtime.

5. **Error message passthrough:** Modified updater:check handler to capture error details and send to UI instead of silent failure. Improves debugging and user feedback.
