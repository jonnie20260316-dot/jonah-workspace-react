## Summary

Replaced the fixed hero header with a modern floating toolbar system. This opens the top of the canvas for more content and consolidates frequent controls into a compact, always-accessible strip.

## What Changed

### HTML Structure
- **Removed:** Fixed hero header (~52 lines) with language toggle, snap button, date nav, gear menu
- **Added:** Three new overlay components:
  1. `.floating-toolbar` — 3-zone layout fixed at `top: 12px`, pointer-events: none container
  2. `.dropdown-panel` — 320px overlay with icon filter rail + history content body
  3. Gear menu flyouts with hover-to-expand submenus

### CSS Changes
- **Removed:** ~100 lines of hero-specific CSS (hero header, hero button, icon-rail)
- **Added:**
  - `.floating-toolbar` — `position: fixed`, `pointer-events: none`, 3 pill buttons (`.ft-pill`)
  - `.ft-save` — hidden by default, visible on `.dirty` class
  - `.dropdown-panel` — fixed overlay, scrollable history content
  - `.gear-category` + `.gear-submenu` — 4 groups with hover-expand animation
  - Grid: `4 columns` → `2 columns` (minmax(0, 1fr) + archive width)

### JavaScript Changes
- **Removed:** `heroCollapsed` module-level variable
- **Refactored:** `topbarCopy()` — deleted 12 hero-specific copy lines
- **Added:** `dropdownOpen` state variable + click handlers for hamburger + history icon tabs
- **Updated:** `renderHistorySidebar()` to respect `dropdownOpen` state
- **Event bindings:** All listeners re-wired for new toolbar structure
